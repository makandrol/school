## Як створювати новий урок

### Крок 1 — Прочитай контекст

Для **будь-якої** групи, завжди прочитай:
1. `shared/context.md` (цей файл)
2. `<папка_групи>/curriculum.md` — курикулум (що вже пройдено, що далі)
3. **ВСІ `summary.md`** в папках уроків групи — щоб розуміти повний контекст

Також прочитай відповідні **шаблони**:
- Робототехніка: `shared/templates/presentation_robotics.html`, `shared/templates/google_classroom_robotics.md`
- 3D: `shared/templates/presentation_3d.html`, `shared/templates/handout_3d.html`

### Крок 2 — Створи файли уроку

**Робототехніка** — створи папку `lesson-XX/` з:
| Файл | Опис | На сайті? |
|------|------|-----------|
| `presentation.html` | Презентація (самодостатній HTML, навігація стрілками) | ✅ |
| `google_classroom.md` | Текст для Google Classroom | ✅ |
| `summary.md` | Підсумок для наступного агента | ❌ |

**3D-моделювання** — створи папку `lesson-XX/` з:
| Файл | Опис | На сайті? |
|------|------|-----------|
| `presentation.html` | Презентація (самодостатній HTML) | ✅ |
| `handout.html` | Роздатковий матеріал для друку (HTML з SVG) | ✅ |
| `summary.md` | Підсумок для наступного агента | ❌ |

### Крок 3 — Онови manifest.json

Додай урок в масив `lessons` відповідної групи в `manifest.json`:

Для робототехніки:
```json
{
  "n": 23, "title": "🎯 Назва уроку", "date": "17.03.2026",
  "published": true,
  "files": ["presentation.html", "google_classroom.md"],
  "makecode": [
    {"label": "Назва проєкту", "url": ""}
  ]
}
```

Для 3D-моделювання:
```json
{
  "n": 6, "title": "🎨 Назва заняття", "date": "17.03.2026",
  "published": true,
  "files": ["presentation.html", "handout.html"],
  "tinkercad": [
    {"label": "Назва моделі", "url": ""}
  ]
}
```

**Поля `makecode`/`tinkercad`:** URL заповнюється пізніше вчителем. Залишай `"url": ""`.

## Загальні правила для матеріалів
- Мова: українська
- Стиль: дружній, зрозумілий для дітей відповідного віку
- Для 3-4 класу: прості слова, багато візуальних описів, короткі речення
- Для 5 класу: можна складніші терміни, але з поясненнями
- Для 5-7 класу: можна ще складніші поняття, але все одно пояснювати
- Кожен урок починається з інструктажу з БЖД (для робототехніки — нагадування)
- Для робототехніки: **описуй все через MakeCode блоки** (не pseudocode, не raw JavaScript)

## 📊 Формат презентацій — самодостатній HTML

Презентації створюються **одразу як .html файли** — без проміжних форматів чи конвертації.
Вчитель відкриває .html в браузері → стрілки ← → для навігації → F11 повний екран.

### Структура HTML-презентації:
- Самодостатній HTML з вбудованими CSS та JS (без зовнішніх файлів!)
- Кожен слайд — `<div class="slide">`, навігація стрілками через JS
- Перший слайд — титульний з `class="slide lead"` (градієнтний фон)
- Колірна схема: **фіолетова** (#667eea → #764ba2) для робототехніки, **зелена** (#00b894 → #00cec9) для 3D
- JS для навігації: стрілки ← →, пробіл, клік мишкою, Home/End
- Лічильник слайдів внизу
- Дивись шаблон: `shared/templates/presentation_robotics.html` або `presentation_3d.html`

## 📄 Формат роздаткових матеріалів — самодостатній HTML

Роздаткові матеріали створюються **одразу як .html файли** для друку.
Вчитель відкриває .html в браузері → Cmd+P → друк.

### Обов'язково в кожному handout.html:
- Самодостатній HTML з вбудованими CSS (без зовнішніх файлів!)
- `@page { size: A4; margin: 12mm 14mm; }` для правильного друку
- Кожне завдання в `<div class="page">` з `page-break-after: always`
- **1 завдання = 1 сторінка** (для складних завдань — можна більше)
- Перша сторінка: заголовок заняття + нагадування нових інструментів + список завдань
- **Інлайн SVG-картинки** — показують що має вийти в результаті (і проміжні кроки якщо потрібно)
- SVG-картинки мають бути невеликі (max ~320×180px), щоб поміщались на сторінці разом з текстом
- Позначки складності: ⭐, ⭐⭐, ⭐⭐⭐, ⭐⭐⭐⭐
- Дивись шаблон: `shared/templates/handout_3d.html`
