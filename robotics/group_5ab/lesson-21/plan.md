#### 3. ⭐ Завдання 1 — Гра «Реакція» (12 хв)
**Суть:** Micro:bit чекає випадковий час (1-5 секунд) → показує ❤️ → гравець натискає A якнайшвидше → micro:bit показує час реакції в мілісекундах.

**Алгоритм:**
```
on button A pressed:
  clear screen
  pause (pick random 1000 to 5000)    ← випадкова пауза 1-5 секунд
  show icon ❤️
  set старт = running time (ms)       ← запам'ятовуємо поточний час

on button B pressed:
  set реакція = running time (ms) - старт  ← різниця = час реакції
  show number реакція                  ← показуємо результат в мс!
```

**Змінні:** `старт` (момент появи ❤️), `реакція` (результат)

**Розбір:**
- `running time` — це "годинник" micro:bit, рахує мілісекунди з моменту ввімкнення
- Різниця двох моментів = час між ними
- Нормальна реакція людини: 200-400 мс
- Менше 200 = супершвидко! Більше 500 = треба тренуватись

#### 4. ⭐⭐ Завдання 2 — Гра «Вгадай число» (12 хв)
**Суть:** Micro:bit загадує число від 1 до 10. Гравець вгадує — кнопка A збільшує спробу, B зменшує. Shake — перевірити. Micro:bit показує стрілки ↑ (більше) або ↓ (менше) або ✓ (вгадав!).

**Алгоритм:**
```
on start:
  set загадане = pick random 1 to 10
  set спроба = 5                       ← починаємо з середини
  show number спроба

on button A:
  change спроба by 1
  if спроба > 10 → set спроба = 10     ← не більше 10
  show number спроба

on button B:
  change спроба by -1
  if спроба < 1 → set спроба = 1       ← не менше 1
  show number спроба

on shake:
  if спроба = загадане:
    show icon ✓ (happy)
    show string "YES!"
    pause 2000
    set загадане = pick random 1 to 10  ← нова гра!
    set спроба = 5
    show number спроба
  else if спроба < загадане:
    show arrow North                    ← стрілка вгору = "більше!"
    pause 500
    show number спроба
  else:
    show arrow South                    ← стрілка вниз = "менше!"
    pause 500
    show number спроба
```

**Змінні:** `загадане` (таємне число), `спроба` (поточне число гравця)

#### 5. ⭐⭐⭐ Завдання 3 — Гра «Бомба» (9 хв)
**Суть:** Micro:bit — це «бомба»! Показує зворотний відлік (10, 9, 8...), але зупиняється на випадковому числі і «вибухає» 💀. Гравець натискає A, щоб запустити, а потім B, щоб «зупинити таймер» (спробувати «розмінувати»). Якщо натиснув B на правильному числі — врятувався, якщо ні — вибух!

**Алгоритм:**
```
on button A:
  set бомба = pick random 3 to 8      ← "бомба вибухне" на цьому числі
  set рахунок = 10
  set вибухнула = 0

  while рахунок > 0:
    show number рахунок
    pause 400
    if рахунок = бомба AND вибухнула = 0:
      show icon 💀 (skull)             ← ВИБУХ!
      set вибухнула = 1
      show string "BOOM"
      break                            ← вихід з циклу
    change рахунок by -1

  if вибухнула = 0:                    ← дійшли до 0 без вибуху?
    show icon ✓
    show string "OK!"

on button B:
  if вибухнула = 0:
    set рахунок = 0                    ← "розмінували" — зупиняємо відлік
```

**Альтернативний (простіший) варіант:**
```
on button A:
  set час = pick random 5000 to 15000  ← 5-15 секунд
  show icon ❤️                          ← "бомба тикає"
  pause час
  show icon 💀                          ← ВИБУХ!
  show string "BOOM!"
```
Цей варіант — просто таймер, в кого «вибухне» — той програв. Передаємо по черзі!

**Змінні:** `бомба` (число вибуху), `рахунок` (поточний відлік), `вибухнула` (прапорець)
